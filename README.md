# Spatial Learning: Frontend to Full-Stack Engineering Program

A structured, project-based curriculum that trains frontend developers into strong full-stack engineers using **spatial learning** — understanding not just *what* code does, but *where* it lives, *how* it connects, and *why* it's organized that way.

## Program Philosophy

Most tutorials teach you to type code. This program teaches you to **see systems**.

Every concept is anchored in physical and spatial metaphors:
- Where does this code run? (browser, server, database)
- What layer of the stack are we in?
- How does data travel between layers?
- What are the trust boundaries?
- Why is this folder here and not somewhere else?

You will build **5 real, production-ready applications**, each more complex than the last. Each app is independently deployable, GitHub-ready, and demonstrates specific skills to employers.

## Curriculum Overview

| Level | App | Core Concepts | You Will Build |
|-------|-----|--------------|----------------|
| **1** | DevPulse (Mood Tracker) | React, TypeScript, Express, REST, Git | Single-page app with simple backend |
| **2** | TaskForge (Project Manager) | PostgreSQL, CRUD, API design, deployment | Full database-backed application |
| **3** | VaultNote (Secure Notes) | Auth, JWT, protected routes, security | Authenticated multi-user app |
| **4** | DataDash (Analytics Dashboard) | State management, testing, middleware, performance | Scalable architecture app |
| **5** | CollabBoard (Project Hub) | Advanced UX, CI/CD, monitoring, modular backend | Production-grade portfolio piece |

## Prerequisites

- Comfortable writing HTML and CSS
- Basic JavaScript knowledge (variables, functions, arrays, objects, loops)
- A computer with macOS, Windows, or Linux
- Curiosity and willingness to be confused temporarily

## How to Use This Program

1. Start with the [Setup Guide](./00-setup-guide/) to configure your development environment
2. Read the [Curriculum Overview](./00-curriculum-overview/) to understand the full journey
3. Work through each level sequentially — they build on each other
4. **Do not skip levels.** Each one introduces concepts the next level assumes you know.
5. At each level, build the app yourself. Reading is not learning. Building is learning.

## Technologies Covered

**Core Stack (used at every level):**
- React.js + TypeScript
- Node.js + Express
- Git + GitHub
- VS Code

**Introduced progressively:**
- PostgreSQL (Level 2) — persistent data storage
- JWT Authentication (Level 3) — user identity and security
- Redux Toolkit (Level 4) — complex state management
- Vitest + React Testing Library (Level 4) — automated testing
- GitHub Actions (Level 5) — CI/CD pipelines
- Sentry (Level 5) — error monitoring

## Growth Tracking

Each level ends with:
- A deployed, working application
- A list of skills the app demonstrates to employers
- Guidance on how to explain the project in interviews
- Clear articulation of architectural understanding gained
